Where each tab sits in the architecture
HTTP / API Flow
All 10 Levels
The end-to-end request journey travels through every architecture level — Client → Edge → LB → Gateway → Backend → Cache → DB. HTTP is the thread that connects them all.
OSI Model
Cross-cutting — not a level
OSI is the underlying network protocol stack that every architecture level uses to move data. L1–L4 (bits→TCP) operate inside Levels 2–3; L5–L7 (session→HTTP) are used by Levels 1–5. OSI describes how data travels, not what your app does.
TCP/IP Flow
Level 2 (Network/Edge) + Level 3 (Traffic Management)
TCP connection setup and TLS termination happen at the CDN edge and load balancer — before your application code sees the request. Managed entirely by network infrastructure.
DNS Resolution
Level 2 (Network / Edge)
DNS is a network service that resolves domain names to IPs before any TCP connection starts. It runs at OSI L7 (DNS application protocol) over UDP/TCP, but is an infrastructure concern sitting in the Network/Edge layer.
Auth Flow
Level 4 (API Gateway / Security)
JWT validation and OAuth token exchange are enforced at the API Gateway — the boundary between infrastructure and your services. The client (L1) stores the token; the backend (L5) may check fine-grained permissions; but the primary enforcement point is L4.
