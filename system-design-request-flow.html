Where does the API actually live? The API contract — the URL structure, HTTP methods, and request/response shapes — is defined and implemented in the Backend Server (step 06). Steps 05–08 form the API boundary (marked green above): the Gateway enforces access policies on it, the Cache accelerates its responses, and the Database persists its data. Steps 01–04 are pure infrastructure routing traffic toward your API — they don't know or care what endpoints exist.

Server vs Domain: The server (an EC2 instance, a Droplet, a container) is a machine that runs 24/7 with your code listening on a port. It gets an IP address (e.g., 54.239.28.85) assigned by the cloud provider. The domain (api.example.com) is bought separately from a registrar (Namecheap, Cloudflare) and linked to the server via a DNS A record. IANA → Regional Registries (ARIN, RIPE) → Cloud providers manage IP space; ICANN → Registrars manage domain names.
